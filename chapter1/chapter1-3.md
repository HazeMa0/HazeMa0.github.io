## 第三幕 数字运算的世界

在上一节，我们已经完全理解了我们的第一个C语言程序。这一节，我们争取自己写出一些程序来。

既然是计算机，计算的功能当然是必要的。接下来我们将介绍各种数据类型和运算。

### 并非是无穷大

在上一节的函数中我们知道，函数最开头必须写明其返回值的类型，在那时我们已经知道了`int`类型可以存储整数。

但是，任意大/任意小的整数都可以被`int`存储吗？这个问题很好反驳，因为存储器的容量是有限的，我们总可以找到一个足够大的数字来塞满整个存储器。那么，`int`最多可以存储多大/小的数字呢？

> [!hw]
> **int类型的梦魇：2147483648（2的31次方，记为2^31）**
>
> 将以下代码放入main函数，观察打印的值。
>
> ```c
> int i = 2147483648;
> printf("%d\n", i);
> ```
>

你很快发现了不对劲！i的值明明是`2147483648`，为什么打印出来的是`-2147483648`？？？这件事打破了你对计算机擅长计算的认知……你想象到，也许你的程序有一天会因为产生了这个数字而导致程序的错误，这实在是让人无法安心。但是你也不必悲观，因为我们马上就要解释产生这样糟糕结果的原因。

> [!question]
> **最大值溢出造成的危机：千年虫、2038年问题**
> 
> 当20世纪的人们迎接新千年的到来时，他们不仅有见证历史的欣喜，还有对计算机千年虫问题的担忧。什么是千年虫问题？原来，在当年存储设备寸土寸金的年代，人们为了占用更小的体积，使用年份的最后两位表示年份。这种习惯被延续下来，直到逼近2000年，人们突然意识到，计算机系统的年份将突然从99跳到00，这将导致依赖于系统时间的程序很容易崩溃。即使严加防范，最后千年虫这个巨大的Bug还是给人们造成了不少的麻烦。
> 
> 与此类似的还有32位linux系统的2038年问题。linux通过距离1970年1月1日的秒数来表示时间。这个数值将在2038年到达`2147483638`……你刚才已经见识到了这意味着什么。好在所有新的计算机设备和手机都是64位系统，将使用64位二进制数来表示时间，这个问题应该在2038年不会造成太大的危害。
>

&emsp;

> [!question]
> **我的电脑是32位还是64位？**
>
> 这个问题通过简单的百度就能找到很好的查询办法。
>

众所周知，在计算机中，我们使用二进制表示数据。那这样的1个0或者1就表示了两种可能性，我们称之为1个`位（bit）`。我们约定，8个位组成一个`字节（Byte）`，2^10=1024个字节组成一个KB，1024个KB组成一个MB，以此类推，GB，TB……

聪明的你肯定意识到了我强调2147483648是2^31的意义。事实上，int的表示范围是：

-2147483648（-2^31） ~ 2147483647（2^31 - 1）

这就很巧了。这意味着，在计算机中，给int最大值+1后，变成了int的最小值！这就像是计算机中的数字围成一个圈，最大值的下一个就是最小值。这样的计算系统我们在数学称之为`模运算系统`。

很容易计算，int类型可以表示`2^31 - 1`个正数，`2^31`个负数，还有一个0，总共2^32个数。很容易猜想到，**int类型使用4个字节来存储。**

那我要存储更大的数据要怎么办呢？计算机提供了`long long`类型：

```c
long long i = 18446744073709551615; // 2^64 - 1
// 注意，打印long long类型时，%d换为%lld
printf("%lld\n", i); // 结果：18446744073709551615
i++; // 这是一种简便写法，意思是 i = i + 1;
printf("%lld\n", i) // 结果：-18446744073709551616
```

如你所见，**long long类型占用8个字节的空间**。在上面的代码中，我们用到了`注释`，他们在一行的最后，以`// `开始，使用中文和英文书写来便于他人和你自己理解代码。写一些注释是很好的编程习惯。

那更大的数字呢？如果你只存储正数和0，你将获得更大的最大值的两个新类型：

```c
unsigned int uint_max = 4294967295; // 2^32 - 1
unsigned long long ull_max = 18446744073709551615; // 2^64 - 1
```

> [!tip]
> **变量的名称应当易于理解**
>
> 代码不仅要用于生成程序，也要接受很多人的阅读。如果你使用a,b,c等等没有意义的变量名称，那过10分钟你就会没法完全理解自己的代码。所以像上面一样，用max暗示这是最大值，代码就会很好理解。
>
> 将心比心一下，如果C语言的int换成y，long long换成t，你会容易理解和记忆吗？
>

### 不可能没有小数吧

当然有！由于计算机存储小数的方式，我们将小数叫做`浮点数`。

> [!hw]
> **如果你是计算机科学家，你会怎样存储小数？**
> 
> 我们一起来想象一种存储小数的类型叫做**cute**（现实中不存在，小小的也很可爱！），占用4个字节，也就是32位。你会怎么设计存储小数的规则？
> 
> 很自然，我们想象到一种办法，就是一半用来存储小数点前面的数，一半用来存储小数点后面的数。这种存储方法现实里还真有，叫做`定点数`。这种办法的问题其实不小：很大的但小数位没几位的 和 很小的但小数位后面很长的 都存不进去。空间被浪费了。
>
> 聪明的你马上想到了新的办法：我把32位分成三部分，第一部分只要一位确定正负，第二部分存储去掉小数点的值，第三部分存储小数点的位置不就行了吗？恭喜你，你和计算机行业的先驱的想法是一致的，这种存储方法就是`浮点数`（浮动的小数点）。
>
> 只需要简单的百度，你就可以了解到下面介绍的float类型和double类型具体是如何使用这种方法存储的。
>
> 在计算机发展的早期，存储空间寸土寸金，计算机科学家为了存入更多的数据费劲了心思。我们今天就享受着他们的研究带来的美好生活。世界破破烂烂，他们缝缝补补。
>
> 就用刚才已经介绍过的int类型来举例。我们来一起设计一种int的具体存储方式。一种自然的想法就是最高位表示符号正负，剩下的就把十进制转成二进制存起来就行了。但是事实其实不是这样，因为如果这样0将有`1后面一堆0`和`0后面一堆0`两种表示方式，为了减少复杂和存储哪怕多一个数字，计算机科学家发明了`补码`这个概念，你可以通过百度很快了解它的存储方式。
>

在C语言中，浮点数有两种类型：**一种是占用4个字节的`float`类型，一种是占用8个字节、更加精确的`double`类型。float可保证7位十进制有效数字，double可保证16位十进制有效数字。**

> [!hw]
> **以身试法！**
>
> 测试结果。
> ```c
>     float f1 = 1.234567898; // 11 digits
>     float f2 = 2.345678999; // 11 digits
>     // 我们使用%f输出float
>     printf("%f\n", f1 + f2);  // should be 3.580246897, but is it the result?
>     double d = 123.56789000000000000000000001; // 30 digits
>     // 我们使用%lf输出double
>     printf("%lf\n", d);  // what is the result?
>     return 0;
> ```

### 加减乘除，还有呢？

数字之间当然可以进行计算。除了我们已经见到过的加减法，还有一些新奇东西，下面我们约定一个式子里的数据表示相同类型的变量：

1. 自增运算（`i++`、`++i`）：这两个都是对i加1的意思。但是有什么区别呢？给出以下代码，去试试吧！

```c
int num1 = 3;
printf("%d\n", num1++); // 3
printf("%d\n", num1);   // 4
printf("%d\n", ++num1); // 5
```

你发现了一个有趣的现象：当需要对这两兄弟求值时，`i++`是先返回个i，然后赋值i = i+1；`++i`是先赋值i = i + 1。再返回新的i。当我们不关心这个式子的返回值时，那这两个都是一样的。

> [!question]
>
> **i怎么可能等于i+1？？？**
>
> 不不不，`i = i + 1;`的意思是把等号右边计算的结果赋予i，这叫赋值语句。`int i = 2;`就是把定义i和给i赋值放在了一起的简便写法。
>
> **为什么要定义一个变量？**
>
> 事实上，有的语言确实不需要定义变量。你想用只需要写一个新名字，系统会帮你自动生成。但是这也会带来一些麻烦：如果我不小心拼写错了一个变量，系统不但不给我报错，还生成了一个新的变量。这是一段python语言的代码，第二行的sheep和shep微小的区别会酿成大错：
>
> ```python
> sheep = 0
> shep = sheep + 3
> print(sheep)  # 结果是0
> ```

2. 乘法运算（`a * b`）

3. 整除运算（`int1 / int2`）

> [!warning]
>
> **整数和整数的运算结果必须是整数！！！**
>
> 这意味着 `2/3 == 0` 而不是0.66666667。（显然C语言的整除只保留整数部分而不是四舍五入。）
>
> 我们教给大家如何进行类型之间的强制转换。例如，`(double)2`将2转化为小数；`(int)7.5`将7.5转化为整数。
>
> 要进行除法运算，必须将需要将被除数或者除数转化为小数。
>
> ```c
> // 这三个才能算出0.6666……
> printf("%lf\n", (double)2 / 3);
> printf("%lf\n", 2/ (double)3);
> printf("%lf\n", (double)2 / (double)3);
> ```

4. 货真价实的除法运算（`float1 / float2`）

5. 取余运算（`int1 % int2`）：取余就是求余数的意思。例如：`7 % 3 == 1`。

> [!warning]
>
> **`==`和`=`的区别！**
>
> 在C语言中，`=`只用于赋值语句，意思是将等号右边的值赋予左边：
> ```c
> i = 3 + 2;  // 正确，给i赋值(3 + 2)也就是5
> i = i + 1;  // 正确，给i赋值(i + 1)也就是(5 + 1)也就是6
> 7 = 7;  // 报错，怎么能给7赋值7呢？
> 6 = i;  // 报错，怎么能给6赋值i也就是6呢？
> ```
> 在C语言中，`==`才用于判断是否相等，相等为1，否则为0：
> ```c
> int i = 6;
> printf("%d\n", (2 == 2));  // 打印1
> printf("%d\n", (i == 9));  // 打印0
> ```

6. 二进制右移符号：(>>) (10>>1 == 5)  （10二进制表示为1010，右移1位为101也就是5）

7. 二进制左移符号：(<<) (10<<1 == 20) （10二进制表示为1010，左移1位为1010也就是20）

### `printf`和`scanf`

关于`printf`，我们已经用过多次，却从来没有系统地讲解过。下面我将给出一些示例，来帮助大家理解：
```c
printf("res is: 123\n");          // 123
printf("res is: %d\n", 456);      // res is: 456
printf("res1 is %d, res2 is %f.\n", 123, 456.789);      // res1 is: 123, res2 is 456.789.
printf("res is: %d\n", 1.9);      // res is: 1717986918  不使用对应的符号会产生错误的值！
```

|数据类型 | 与之匹配的格式字符 |
| :--: | :--: |
| `int`  | `%d`（十进制）、`%o`（八进制）、`%x`（十六进制） |
| `unsigned int`（也称`unsigned`） | `%u`（十进制） |
| `long long` | `%lld`（十进制）、`%llo`（八进制）、`%llx`（十六进制） |
| `unsigned long long` | `%llu`（十进制） |
| `float` | `%f`、`%.5f`（第二个的意思是保留五位小数）|
| `double` | `%lf`、`%.9lf`（第二个的意思是保留九位小数）|
| 后面介绍的`char` | `%c`|
| 后面介绍的`char*` | `%s`|

关于scanf，它的作用是从屏幕获取你输入的值：
```c
scanf("请输入两个数字：%d %d", &a, &b);
```
> [!warning]
>
> **scanf的参数前面要加&！！！**
>
> 这是为什么呢？实际上，在一个变量前面加上&表示这个变量的地址也就是门牌号。
>
> 你想想，printf要打印a，它需要的是a的值，所以写`a`；scanf要把读取到的数据给a，它当然需要的不是a的值而是a的门牌号`&a`了。
