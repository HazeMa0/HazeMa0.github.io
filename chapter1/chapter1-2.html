第二幕 开天辟地的尝试

第二幕 开天辟地的尝试

消化一下

在上一节，我们已经成功运行了以下C语言程序：

#include <stdio.h>

int main(void)
{
    printf("Hello world!\n");
    getchar();
    return 0;
}

下面，我们将对这段代码进行简单的解释。

#include <stdio.h>是什么意思？这个问题我们放到最后解释。

下面一整段代码的开头是int main(void)，后面所有的代码都被一对大括号括住了。这是什么意思？

原来，main是C语言中的函数。是函数，自然会有自变量和函数值。在程序设计中，我们将自变量称作参数（parameter），而把函数值叫做返回值。其中，参数就列在函数名称后面的那一对小括号里面，返回值就写在函数里的return语句里面。

为了解释这些概念，我们来一起分析一下上面的代码：

  1. 函数名：main
  2. 参数：void（代表没有参数）
  3. 函数本体：就是大括号里面的那三行代码。
  4. 返回值：恒为0。

这时候你已经产生了很多疑问，比如：

1. 返回值为什么要设置为0？

答：这是一种约定。对于main函数，我们约定返回0表示程序正常结束，非0表示程序没有正常结束。

2.main前面的int是什么意思？

答：这个int表示了返回值的类型为整数（Integer）。关于类型的更多知识我们会在下一节介绍。

3.函数名字可以不叫main吗？

答：当然可以，但是在这里不行。默认情况下，main函数是C语言程序开始的地方。如果没有main函数，无法成功编译。

4.感觉计算机里面的函数和数学里的函数好不一样……

答：当然是的。在数学里面，我们只关心函数的自变量和因变量。也就是说，如果两个函数的自变量和因变量完全相同，这两个函数就相同。但是在计算机领域并非如此。别忘了，我们在返回0之前，还做了打印Hello world的工作。这些因素被我们称为函数的副作用，这就是两者不同的地方。

现在你终于理解了整个程序的运行过程：由系统调用你的main函数作为开始，而且给main函数不给予任何参数。main函数开始执行：第一行你已经猜到是在打印Hello world这段话；第二行先不管他；第三行返回0代表程序正常结束。

[!hw] return 0;之后是什么？

如果一个函数已经返回了值，在数学领域这一次求值已经结束了。在计算机领域呢？你可以在return 0的下一行添加一句新的printf("handsome man\n");，观察是否可以打印这句话。

细嚼慢咽

现在我们来看看函数本体的前两行。

在C语言中，每个语句都用英文分号;作为结束，反而换行是不需要的。也就是说，你可以把这三句语句写在一行里面！但是我们强烈建议你不要这样做：这样会导致代码难以阅读。

观察第一句：

printf("Hello world!\n");

其实，这句话是在进行一次函数调用。也就是说，printf也是一个函数，而就像你实现了main函数一样，系统已经提前实现了printf函数。这个函数的作用显而易见，就是打印你给出的这段话，在计算机里我们称之为字符串（String，一个个字符穿起来就是一段话）。

那，我们给printf的参数是什么呢？就是括号里的字符串。那它的返回值是什么呢？能提出这个问题真的很棒。

接下来，我们来看看printf返回的值是什么类型的。在VSCode里面，按住ctrl键，点击代码中的printf，你将进入早已耳闻的stdio.h里面。里面是这样写的：

int printf (const char *__format, ...)
{
  register int __retval;
  __builtin_va_list __local_argv; 
  __builtin_va_start( __local_argv, __format );
  __retval = __mingw_vprintf( __format, __local_argv );
  __builtin_va_end( __local_argv );
  return __retval;
}

尽管我们还无法完全理解这段代码，但是你已经发现了奥秘：来自神秘位置的stdio.h里面竟然有printf的实现代码！你现在明白了，原来要使用printf，必须要加上#include <stdio.h>来让系统找到printf。

你很快就可以理解，printf返回值的类型是int，printf接受的第一个参数的类型是const char *（这就是字符串的类型）。

那么，printf返回的int值究竟去哪里了呢？显而易见，它被我们放弃了。我们根本没有关心过printf的返回值。

[!question] Q：printf的返回值不应该是hello world吗？它打印了这段话呀？

A：当然不是。如你所见，printf返回值的类型是int，根本不是字符串。事实上，打印的效果是printf在执行过程中勾结显示器为你打印值导致的，这是它的副作用。返回值是printf函数里的return语句决定的。

 

[!hw] printf好可怜，我要关心它的返回值！

很好！我就知道你会来关心的。作为真正使用函数调用的第一个例子，我来告诉你如何获得它的返回值。

有一个很简单的想法：我们只需要将printf的返回值给另一个printf打印不就行了！将第一行printf换为如下代码：

printf("%d", printf("Hello world!\n"));

外面的printf有两个参数：“%d”表示要把%d换成第二个参数的值。第二个参数也就是printf("Hello world!\n")，它的值是多少呢？系统会先去运行这个printf，获得它的返回值然后交给外面的printf。这样，我们就获得了第一个printf的返回值。

屏幕上的结果如下：

Hello world!
12

Hello world!是哪里来的？是系统计算里面的printf的时候打印的。12是什么意思？是printf("Hello world!\n")的返回值。

12是什么意思呢？其实，是"Hello world!\n"的字符数量。其实，\n代表一个字符，它代表了回车。删掉\n，你会发现两行会粘在一起。其实，键盘上的CTRL、alt、Tab……都是字符！这真是个惊天大秘密！现在数一数，是不是12个？

最后，只要理解getchar();就可以了。按住Ctrl键点击代码中的getchar，我们可以看到它也返回int值，不需要参数。事实上，getchar的作用是从你这里讨要字符。假设没有getchar，系统完成printf后就会return 0，这个程序就结束了。结束了是不是应该关掉终端？这样会导致程序从打印到结束就是一闪而过，你根本看不见。有了getchar，他会找你讨要字符后再结束程序，这样我们就可以看到结果了。

[!note] 有的IDE（例如Visual Studio）不需要getchar。它不会自动关闭终端窗口。

终于，我们理解了这段代码，可真不容易呀！
