第二幕 开天辟地的尝试（全新内容追加）

[!note]

本幕借鉴了CS61A: SICP的课程引入方式，并在此原则上进行重构。

C语言也有函数

在上一节，我们已经成功运行了第一个C语言程序。但那个程序的解释我们暂且按下不表，在本幕的最后我会向你解释整个程序。

现在，我们来学习一个C语言一个很重要的概念——函数。

和数学里的函数一样，C语言里面的函数也得有自变量和函数值。不过，在计算机领域，我们一般将函数的自变量叫做参数（parameter），函数值叫做返回值（return value）。

很自然的一个问题：我们怎么创造函数？其实，就在上一幕，我们已经创造过一个函数了——main函数。现在我们把上一幕的main函数拉出来溜溜：

int main(void)
{
    ... // 这部分内容暂且按下不表
    return 0;
}

在C语言中，程序从调用main函数开始。系统会一句一句地执行main的大括号里面的内容，直到遇到return。

[!hw] 这意味着什么？

你可以在return 0;的下一行添加一句printf("handsome man\n");，观察是否可以打印这句话。

显然，return 0;表示这个函数的返回值永远是0。既然已经安安稳稳执行到了这一句，其实return 0就是用和系统约定好用0来报个平安的意思了。

使用函数

更自然的一个问题：我们如何使用函数？现在新建另一个文件addsub.c（还是那句话，如果你想半年后找到以前的代码就不要在一个文件里改来改去），编写如下代码：

// 使用//表示单行注释，C语言会无视每行//后面的所有内容

#include <stdio.h>

int add(int a, int b);     // 这个叫做函数的声明，这表示其他函数会用到这个函数。最后有分号
int sub(int a, int b);

int main(void)
{
    int res;   // 请你暂且忽略所有的int，我们下一幕再来解释。
    res = add(5, sub(2, add(-1, 3)));     // 显然，这是在进行一个神秘的计算
    printf("%d\n", res);     // 请你暂且忽略这里printf的具体写法，只需知道这是在打印res就行了
    getchar();
    return 0;
}

int add(int a, int b)      // 这个叫做函数的实现，下面是这个函数的具体功能。最后没有分号
{
    return a + b;
}

int sub(int a, int b)
{
    return a - b;
}

这个例子已经足够完整地展示了函数的创造和使用。

首先，由于历史原因，要在一个函数里面使用其他函数，要在函数前面加上其他函数的声明。例如int add(int a, int b);表示这个函数的名字叫add，接受两个参数：a和b。

在后面，我们实现了add和sub。很显然，就是进行了加法和减法作为返回值。

现在，我们重点来关心一下main函数里面最重要的res = add(5, sub(2, add(-1, 3)));这句话。

C语言遇到这句话，它会进行如下的计算：

res = add(5, sub(2, add(-1, 3)));
res = add(5, sub(2,     2     ));
res = add(5,         0         );
res =          5                ;

如你所见，以add(-1, 3)为例，C语言会将-1和3作为两个参数给add。add返回2，2就代替了add(-1, 3)。

那add(5, sub(2, add(-1, 3)))呢？计算机很傻，他知道第一个参数是5，第二个是sub(2, add(-1, 3))。要获得第二个参数的值就要计算sub(2, add(-1, 3))，它的第一个参数是2，第二个参数是add(-1, 3)，要获得第二个参数的值……以此类推。

终于，你理解了函数是如何使用（调用）的。把参数给函数，然后用返回值来代替这个函数。

函数名(参数1, 参数2, 参数3, ...)
            |
            v
          返回值

那很显然，你马上理解了我们hello.c文件里的内容了：

    printf("Hello world!\n");

C语言会把"Hello world!\n"给printf，然后执行printf这个函数，然后用其返回值代替这个式子。而且我们已经可以猜出来，printf系统已经帮我们写好了，而且这个函数在执行时勾结显示器为你打印了字符。

可是，在前面，res = add(5, sub(2, add(-1, 3)));最后变成了res = 5;，这表示给res接走了这个5；printf("Hello world!\n");最后只会变成一个值，那这个值就放在那里无人领养，也就相当于不要了。

[!hw] printf好可怜，我要关心它的返回值！

很好！我就知道你会来关心的。测试如下代码（显然要新建另一个文件，放在main函数里，并且不能在return后面）：

printf("%d", printf("Hello world!\n"));

屏幕上的结果如下：

Hello world!
13

这是怎么回事？其实已经很好理解了：外面的printf有两个参数，第二个参数的值是多少呢？系统会先去执行里面的printf（在这里面打印了hello world），然后用返回值替换掉。最后，外层的printf打印了返回值。事实上，返回值是要打印的字符的数量。"Hello world!\n"不是13个吗？事实上，最后"\n"表示换行，代表一个字符。

和之前的终端找软件一样，gcc去哪里找printf呢？所以才有了最开头的#include <stdio.h>嘛。其实printf和后面的getchar都在stdio.h里面有具体的声明，这样方便gcc去找。

[!hw] getchar：为什么这个程序没有闪退？

你也许一脸懵逼，为什么我写的程序要闪退呢？因为按理来说，你打印了printf，然后return 0，程序结束了，窗口自然应该关闭。所以，为什么程序不是一闪而过就关闭呢？

答案就是getchar()。这个函数表示接受一个字符。当你在终端输入任意字符时，getchar()才会结束，这样就避免了程序的闪退。

终于，我们理解了这段代码，可真不容易呀！
